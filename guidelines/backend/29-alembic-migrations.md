# Alembic Migrations

Use when creating or applying schema migrations. For the repository pattern see `backend/04-infrastructure-layer`. For safe migration strategies (zero-downtime, backward-compatible changes) see `backend/23-safe-migrations`.

---

## Standard commands

```bash
# Generate a migration from model changes
poetry run task autorevision   # alembic revision --autogenerate -m 'autogenerated'

# Apply all pending migrations
poetry run task auhead          # alembic upgrade head

# Roll back all migrations
poetry run task adbase          # alembic downgrade base
```

These are taskipy aliases — add them to `[tool.taskipy.tasks]` in `pyproject.toml` alongside the standard `run`, `test`, and `lint` tasks (see `backend/17-project-setup.md`):

```toml
[tool.taskipy.tasks]
autorevision = "alembic revision --autogenerate -m 'autogenerated'"
auhead       = "alembic upgrade head"
adbase       = "alembic downgrade base"
```

## `env.py` — registering all models

The `alembic/env.py` must import every entity module so SQLAlchemy registers them with `Base.metadata` before autogenerate compares the schema:

**`alembic/env.py`**
```python
from src.infrastructure.db.base import Base
import src.domain.entities  # noqa — registers all models with Base.metadata
```

Without this import, `alembic revision --autogenerate` will generate a migration that **drops every table** (it sees an empty metadata against a populated schema).

---

## `Base` definition

**`src/infrastructure/db/base.py`**
```python
from sqlalchemy.orm import DeclarativeBase

class Base(DeclarativeBase):
    pass
```

Every SQLAlchemy entity inherits from this `Base`. Do not scatter multiple `Base` subclasses across the project — autogenerate only tracks models registered with the single shared `Base`.

---

## Anti-pattern: migration that drops unregistered tables

```python
# ❌ WRONG — entity not imported in env.py
# alembic sees: expected tables = {} | actual tables = {notes, users}
# generates: drop table notes; drop table users
```

```python
# ✅ CORRECT — all entities imported in env.py
import src.domain.entities  # noqa
# alembic sees: expected tables = {notes, users} | actual tables = {notes, users}
# generates: (no changes)
```

## Quick Checklist

- [ ] `alembic/env.py` imports `src.domain.entities` (or equivalent) to register all models
- [ ] All entities inherit from the single shared `Base`
- [ ] Autogenerated migration reviewed before applying — check for unexpected `DROP TABLE` or `DROP COLUMN`
- [ ] Migration applied locally (`auhead`) and tests re-run before pushing
- [ ] Destructive changes (column drops, renames) follow `backend/23-safe-migrations`
